#Data Lake is a central repository to store all types of data at any scale , in the form of files.

# How Spark writes to Data Lake ?

We have a Spark cluster and a data lake .

Job (process sales data)

Spark will create a job divided into multiple , parallel tasks to write multiple, partitioned files parallely to the Data Lake folder.

                        |-Task1-------------------------------> part-000.csv    } (partition files)
                        |- Task2------------------------------> part-001.csv    }
Job (process sales data)|- Task3------------------------------> part-002.csv    }DATA LAKE
                        |-Task4-------------------------------->part-003.csv    }
						
#ACID guarantees on transactions.
Atomicity - All or no changes are written .
Consistency - Data always remains in valid state before and after the transaction.
Isolation - Transaction must run isolated from other processes. Any data written by the writer should not be visible to the Reader till it is committed.
Durability - Once committed , data persists even if system fails.

Data Lakes does not provide ACID guarantees.
In Spark overwrite is a 2 step process. Delete existing + write two new part files.

Challenges with Data Lake.
- Data corruption because of failures - no rollback.
- No data validation while writing. So a column could be written as integer in one transaction and string in another.
- Consistency issues while reading data.
- Handling Batch and Streaming data together is tough.
- No updates / deletes / merge on files.(Difficult to implement GDPR/CCPA compliance).
- Data quality issues.- Schema isn't verified before writing.
                      - Cannot apply checks on data (like in the databases)
- Query performance issues.
- Difficult to maintain historical versions of data.


Delta Lake can help solve these challenges. Apache Iceberg and Apache Hudi that works similarly to Delta Lake.
Delta Lake is an open source storage layer that brings reliability to Data Lakes.
We need to deploy the Delta Lake component in Spark cluster.
Data from a dataframe will be written using delta format and stored as Parquet files along with transaction log.

Save Dataframe in Delta format.

(
dataframe
   .write 
   .format("delta") # like other formats - csv , parquet , etc.
   .save (filepath)
   
)

Suppose we want to store Customer data in delta format inside Customer folder. Spark will create a _delta_log subfolder where the transaction log will be stored.

Write operation 1 (write 2 part files) - > part-000.parquet and part-001.parquet. Since we are creating in delta format Spark will also create a file called
000.json that will be stored in the _delta_log subfolder and contain transaction log data.

Write operation 2 (append 1 part file) part-002.parquet and 001.json in the  _delta_log subfolder.

Write operation 3 (append 1 part file) part-003.parquet and 002.json in the  _delta_log subfolder.

For each write operation part files are written first and then a transaction log file is added in json format to _delta_log folder.
Transaction logs contain the details of every transaction that has been performed on the file.

For each read operation first transaction log files are read and part files are then read based on log files.First each all log files - 000.json,001.json , 002.json
and then read 4 part files based on log information. It contains information like operation and file name.
For Update operation log file will contain 2 operations - Remove and add - Remove part1 and add part4.

Delta Lake features - 
Provides ACID guarantees 
- No data corruption because of failures.
- Data consistency while reading data.

Perform inserts/updates/ deletes.

Schema enforcement and evolution.

Protect data using Time Travel.

Handle batch & streaming data together.

Apply data quality checks.

Performance improvements using statistics.


For Job failure in overwriting data existing files are not deleted in Delta format. Log files contains details of only complete and not 
partial transactions.

from pyspark.sql import SparkSession
from pyspark.sql.types import *
from pyspark.sql.functions import *

spark = (
             SparkSession
			     .builder
				 .appName("DeltaLakeApp")
				 
				 .master("local[4]")
				 .config("spark.dynamicAllocation.enabled", "false")
				 
			#Add package for Delta Lake
			     .config("spark.jars.packages", "io.delta:delta-core_2.12:2.2.0")
				 
			#Add settings to use Delta Lake with Spark session.	 
				 
		        .config("spark.sql.extensions", "io.delta.sql.DeltaSparkSessionExtension")
				.config("spark.sql.catalog.spark_catalog", "org.apache.spark.sql.delta.catalog.DeltaCatalog")
				.getOrCreate()
				
       )
sc=spark.sparkContext

spark
				 
#Create schema for Yellow taxi data.

# Read Yellow Taxis file.

yellowTaxiDF= (
                spark
				    .read
					.option("header" , "true")
					.schema(yellowTaxiSchema)
					.csv("C:\SparkCourse\DataFiles\Raw\YellowTaxis_202210.csv")
					
			)
			
# Print Schema of DataFrame
yellowTaxiDF.printSchema()


# Create database in metastore
spark.sql(""" 
CREATE DATABASE IF NOT EXISTS TaxisDB
""")

# Parquet format : Save DataFrame as a Table.

(
    yellowTaxiDF
	       .write
		   .mode("overwrite")
		   .partitionBy("VendorId")
		   .format("parquet")
		   .option("path", "C:\SparkCourse\DataFiles\Output\YellowTaxis.parquet")
		   .saveAsTable("TaxisDB.YellowTaxisParquet")
)

#Delta format : Save DataFrame as a Table.

from delta import *

(
    yellowTaxiDF
	       .write
		   .mode("overwrite")
		   .partitionBy("VendorId")
		   .format("delta")
		   .option("path", "C:\SparkCourse\DataFiles\Output\YellowTaxis.delta")
		   .saveAsTable("TaxisDB.YellowTaxis")
)
